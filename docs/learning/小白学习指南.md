# 从 BPNet 到 TransChromBP 的小白学习指南

这份指南给两类人：

1. 第一次接触这个项目的人
2. 隔了一段时间回来，想快速重新建立正确认知的人

目标不是让你一次性看完所有旧资料，而是用最短路径建立 **今天仍然正确** 的理解。

## Step 0：先补概念，再看项目

如果你对下面这些词还不熟：

- `QKV`
- `attention`
- `KL / JSD`
- `NLL / MSE`
- `loss weight`
- `seed / ablation / early stop`

不要直接跳进项目代码。

先读：

- [深度学习补课手册.md](深度学习补课手册.md)

它的目标就是让你先把这些名词和本项目里的真实位置对上。

## 1. 再分清四类材料

现在的学习资料不再是“旧教程平铺”，而是四层结构：

1. **补课材料**：先把深度学习基础和实验术语补齐
2. **当前模型材料**：今天真正用于训练、评估、写论文的 `TransChromBP`
3. **实验专题材料**：告诉你每条实验线改了什么、结果怎样
4. **基础 / 历史材料**：解释 BPNet / ChromBPNet 从哪里来

如果你一开始不分层，最容易出现两个问题：

- 只看旧材料，结果和当前主线脱节
- 只看当前模型，结果不知道这些设计为什么会出现

## 4. 小白最容易犯的错误

### 错误 1：只看旧 ChromBPNet 教程，不看当前主线文档

这样会直接和当前实现脱节。

### 错误 2：只盯 Transformer，不学 bias factorization

这个项目的核心问题从来不只是“有没有 Transformer”，还包括“怎么 bias-safe 地训练它”。

### 错误 3：只看 `full`，不看 `debiased`

今天如果不同时看 `full / debiased`，几乎一定会误读实验结论。

### 错误 4：把所有 foundation 线都想成“差一点就成功”

仓库里很多 foundation 线已经有比较明确的 verdict：

- Genos：高质量负结果
- Caduceus：near-null
- NT v2：有信号但不互补
- AlphaGenome：sidecar/reference

### 错误 5：把 no-bias 当成推翻主线的结果

它现在只是 supplementary boundary，不是新的默认主线。

## 5. 你看完后至少应该能回答的问题
