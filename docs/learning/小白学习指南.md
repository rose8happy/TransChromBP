# 从 BPNet 到 TransChromBP 的小白学习指南

这份指南给第一次接触这个项目的人，也给“隔了一阵子回来，想快速重新上手”的人。

目标不是让你一次性看完全部旧资料，而是用最短路径建立**今天仍然正确**的认知。

## 1. 先分清三类材料

现在的学习材料不是一堆平铺文档，而是三层结构：

1. **基础材料**：BPNet / ChromBPNet 的概念和历史背景
2. **当前模型材料**：今天真正用于训练、评估、写论文的 `TransChromBP`
3. **内部复盘材料**：解释项目为什么会一步步变成今天这样

如果你一开始不分层，最容易出现两个问题：

- 只看旧材料，结果和当前主线脱节
- 只看当前模型，结果不明白这些设计为什么会出现

## 2. 最短学习路线

### 第一步：先知道当前主线是什么

先读：

- `TransChromBP当前模型学习总览.md`

这一步只要先记住下面这句话：

> 当前默认主线是 `TransChromBP teacher v2 + center pool`，不是旧教程里的 ChromBPNet。

### 第二步：再补基础概念

然后读：

1. `BPNet模型小白教程.md`
2. `BPNet与ChromBPNet模型对比.md`

这一步要搞懂：

1. profile / count 是什么任务
2. BPNet 为什么强调局部卷积与膨胀卷积
3. ChromBPNet 为什么要拆 bias factorization

### 第三步：回到当前实现

再读：

- `TransChromBP当前模型详解.md`

这一步要建立的新认知是：

1. 当前主干是 `ConvStem + LocalDilatedTower + optional Transformer`
2. 当前显式保留 `bias_branch`
3. 当前必须同时看 `full` / `debiased`
4. 当前默认 readout 是 `center pool`
5. 当前训练里 `debiased_profile_weight=2.0` 很关键

### 第四步：如果想知道项目为什么会这样演化

最后读：

- `reports/transchrombp_internal_design_and_experiment_history_20260331.md`

它会告诉你：

- 早期为什么会产生 shortcut 叙事
- clean matrix 为什么推翻了那条强叙事
- strict compare 为什么最后收口在 `L3 shared-region`
- no-bias 为什么只是 supplementary

## 3. 现在最该记住的 6 个事实

### 事实 1：当前默认模型不是旧版 ChromBPNet

今天的默认主线是：

> `teacher v2 + center pool`

它已经把：

- 卷积局部塔
- 可开关 Transformer
- ChromBPNet-style bias branch
- `full/debiased` 双口径

接成了一套统一框架。

### 事实 2：旧教程仍然有用，但它们主要负责补基础

旧材料能教你：

1. 什么是 profile / count
2. 什么是膨胀卷积
3. 什么是 bias factorization

旧材料不能完整解释：

1. 当前 Transformer 如何接入
2. 为什么 `debiased_profile_weight=2.0` 关键
3. 为什么最终默认是 `center pool`
4. 为什么今天必须看 `full/debiased gap`

### 事实 3：现在不能再把“Transformer 特有 shortcut”当主结论

当前更稳的理解是：

1. Transformer 的收益成立
2. 但 clean revalidation 不支持“Transformer 特有强 shortcut”
3. supervision 是否 bias-safe，比 backbone 是否带 Transformer 更关键

### 事实 4：`stop-gradient` 不是唯一关键开关

今天更应该先记住：

- `debiased_profile_weight=2.0`

### 事实 5：当前默认 readout 是 `center pool`

不要把 `attention pool` 当成默认。  
它是研究性对照，不是当前最推荐配置。

### 事实 6：当前最强 external evidence 是 `L3 shared-region`

如果你想看“我们相对 official 是否真的更强”，今天应该优先看 `L3` 收口报告，而不是只看旧 tutorial baseline。

## 5. 小白最容易犯的错误

### 错误 1：只看旧教程，不看当前模型材料

这样会直接和当前主线脱节。

### 错误 2：只学 Transformer，不学 bias factorization

当前项目最核心的问题不是“有没有 Transformer”，而是“怎么 bias-safe 地把它接进来”。

### 错误 3：只看 full，不看 debiased

今天如果不同时看 `full` / `debiased`，就很难真正理解这个项目。

### 错误 4：把 `stop-gradient` 当成唯一关键开关

当前更稳的证据支持 `debiased_profile_weight=2.0` 更像主效应。

### 错误 5：把 no-bias 当成推翻主线的结果

no-bias 现在只是 supplementary 的 boundary result，不是新的主结论来源。
