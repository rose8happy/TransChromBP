# 深度学习补课手册

这份手册给两个对象看：

1. 你平时不做深度学习，但想读懂这个项目。
2. 你知道一些名词，但一看到 `QKV`、`KL`、`JSD`、`NLL`、`early stop`、`selector` 就容易断线。

它不追求教材式推导，只追求一件事：

> 让你能把项目里的代码、配置、实验和这些概念对上号。

如果你只记一句话，请记这个训练闭环：

> 把 DNA 变成数字输入，交给模型做 profile/count 预测，用 loss 告诉模型哪里错了，用 validation/test 检查它是不是在学真正的信号，最后按选模规则保留最值得汇报的 checkpoint。

## 1. 先把训练闭环看懂

一个深度学习项目通常都在做这 6 步：

1. 准备输入和标签  
   这里的输入是 DNA 序列，标签是 ATAC 信号。
2. 把输入送进模型  
   模型会输出 profile 和 count 两类预测。
3. 用 loss 衡量“预测和真实值差多远”  
   差得越远，loss 越大。
4. 反向传播更新参数  
   模型会根据 loss 的方向调整自己。
5. 在 validation 上做中途体检  
   看是不是只会背训练集，还是在真正变好。
6. 在 held-out test 上做最终判定  
   这一步才是“这条线到底算不算成立”的最终证据。

放到本项目里，对应关系大致是：

| 问题 | 本项目里的入口 |
|---|---|
| 输入是什么 | `ChromBPNetBigWigDataset` 与 data config |
| 模型长什么样 | `vendor/transchrombp/transchrombp/models/transchrombp.py` |
| loss 怎么算 | `vendor/transchrombp/transchrombp/training/train_ddp.py` 里的 `compute_losses()` |
| validation 怎么聚合 | `train_ddp.py` 里的 `run_validation()` |
| checkpoint 怎么选 | `trainer.best_metric` + `select_best_epoch.py` |
| held-out test 怎么做 | `vendor/transchrombp/transchrombp/evaluation/evaluate_checkpoint.py` |

## 2. 模型部件：先懂直觉，再看代码

| 概念 | 一句话直觉 | 最小公式 / 伪公式 | 在本项目中的例子 |
|---|---|---|---|
| `token` / `embedding` | 把原始输入切成模型能消费的向量单位 | `x -> embedding(x)` | DNA one-hot、卷积后 token、foundation token 都是在做“把序列变成向量” |
| 卷积 `conv` | 用一个小窗口在序列上滑动，找 motif 和局部模式 | `y[i] = Σ w[k] * x[i+k]` | `ConvStem` 先抓局部 motif |
| 膨胀卷积 `dilated conv` | 看得更远，但不把参数量拉爆 | `x[i + d*k]` | `LocalDilatedTower` 保留 BPNet 风格局部上下文建模 |
| 残差连接 `residual` | 允许“在旧表示上做小修正”，训练更稳 | `y = x + f(x)` | dilated block、部分 adapter 都是残差思路 |
| Transformer | 让每个位置都能参考其他位置的信息 | `y = Attention(x) + MLP(x)` | `SequenceTransformerEncoder` 是当前 backbone 的长程部分 |
| `Q/K/V` | `Q` 是“我想找什么”，`K` 是“我这里有什么”，`V` 是“真正要搬过去的内容” | `Attn(Q,K,V)=softmax(QK^T/sqrt(d))V` | 你可以把某个碱基位置的表示理解为“拿自己的 query 去全序列找相关位置，再把这些位置的信息加回来” |
| `attention` | 给不同位置不同权重，而不是一视同仁平均 | `weight = softmax(score)` | Transformer 内部做长程配对；`attention pool` 读出头也是“学权重再聚合” |
| `pooling` | 把一串位置级特征压成一个更短或更粗的表示 | `mean/max/center pool` | `count_pool_mode=center` 是当前默认读出；`attention pool` 是研究性对照 |

### 2.1 QKV 到底应该怎么理解

如果把一条 DNA 序列想成一排座位：

- `Q`：当前位置想问的问题  
  例如“我附近有没有和我一起决定开放性的另一个 motif？”
- `K`：其他位置手里拿着的标签  
  它告诉你“我这个位置可能和值得你关注的模式有关”。
- `V`：其他位置真正提供的内容  
  不是“我值得不值得看”，而是“你看了之后到底拿走什么信息”。

所以注意力不是“平均看全序列”，而是：

> 每个位置按自己的问题，去挑别的位置的信息来用。

## 3. loss 和指标：它们分别在回答什么

| 概念 | 一句话直觉 | 最小公式 / 伪公式 | 在本项目中的例子 |
|---|---|---|---|
| `NLL` / `negative log-likelihood` | 真实答案如果被模型赋很小概率，就要罚它 | `loss = -log p(target)` | profile 分支常用 multinomial NLL，衡量 profile 形状像不像 |
| `MSE` | 预测值和真实值差多少，就平方惩罚多少 | `loss = (y - y_hat)^2` | count 分支用 MSE，通常在 `logcount` 空间上算 |
| `KL` 散度 | 用方向性方式衡量两个分布差多少 | `KL(P||Q)=Σ P log(P/Q)` | 直觉上是“用 Q 来描述 P 亏了多少信息” |
| `JSD` | 把两个方向的 KL 变成更对称、更稳的分布距离 | `JSD(P,Q)=0.5 KL(P||M)+0.5 KL(Q||M)`，`M=(P+Q)/2` | `peak.profile_target_jsd_full_mean` 是现在经常拿来选模和汇报的主指标 |
| `loss weight` | 告诉训练器“哪类错误更值得优先修” | `total = a*loss1 + b*loss2 + ...` | 当前默认是 `1.0*full_profile + 0.1*full_count + 2.0*debiased_profile + 0.0*debiased_count` |
| `early stopping` | 如果 validation 很久不进步，就别继续烧卡 | “连续 N 次没变好就停” | `early_stop_patience` 控制这件事 |
| `checkpoint selector` | 不是所有 epoch 都保留，要挑一个最值得汇报的 | `argmin(metric)` 或 `argmax(metric)` | 当前越来越强调按 peak JSD，而不是只看总 loss |

### 3.1 为什么这个项目里既看 loss，又看 JSD

因为 loss 和最终汇报指标不是一回事。

- loss 是训练时的“优化代理目标”。
- JSD 是验证/测试时更贴近“profile 形状像不像”的指标。

所以一个常见现象是：

> 训练总 loss 最低的 epoch，不一定是最终 JSD 最好的 epoch。

这就是最近 `loss balance / selector-only` 实验要回答的问题。

## 4. 实验和训练术语：项目里最常见的名词

| 名词 | 一句话直觉 | 本项目中的例子 |
|---|---|---|
| `seed` | 随机种子；同一配置换不同 seed 是为了看结果稳不稳 | `s42`、`s1234` 是最常见的两条对照 |
| `ablation` | 只拿掉一个部件，看它到底有没有用 | `V2-full` vs `V2-noTF`，只关掉 Transformer |
| `validation` | 训练中间的体检集 | 决定 early stop、best checkpoint |
| `held-out test` | 不参与训练和选模的最终考试 | `evaluate_checkpoint.py` 输出最终主数字 |
| `full` | signal 和 bias 融合后的完整输出 | 更接近最终看到的 ATAC 轨迹 |
| `debiased` | 只看 signal branch 的去偏输出 | 用来检查模型有没有把 bias 当捷径 |
| `profile` | 每个位置上的细粒度形状预测 | 例如峰内哪个位置高、哪个位置低 |
| `count` | 整个 region 的总量预测 | 例如这个窗口整体信号强不强 |
| `bias branch` | 专门建模实验偏置的辅助分支 | ChromBPNet-style bias factorization 是项目主干之一 |
| `shortcut` | 模型学了表面上有用、但科学上不稳的捷径 | 早期曾怀疑 Transformer 特有 shortcut，后被 clean matrix 降级 |
| `external evidence` | 不只在自己的一套 validation/test 上好，而是跨口径仍然站得住 | `L3 shared-region` 是当前最强 external evidence |

## 5. 本项目里最容易卡住的 6 个概念

### 5.1 `profile` 和 `count` 不是一回事

- `profile` 问的是“形状对不对”。
- `count` 问的是“总量对不对”。

所以一个模型可能：

- profile 很强，但 count 差一些；
- 或 count 很稳，但 profile 形状不够准。

这就是为什么项目里总是同时报两类指标。

### 5.2 `full` / `debiased` 不是两个数据集，而是同一次前向的两套输出

同一条 DNA 进模型后，当前主线会显式给出：

- `full`：signal + bias
- `debiased`：只看 signal

这不是“多做了一套评估”，而是方法本身的一部分。

### 5.3 `KL` 和 `JSD` 都是在比较分布，不是在比较单个数

profile 不是一个标量，而是一串位置级概率 / 权重分布。
所以 `KL/JSD` 更像是在问：

> 预测出来的“峰形状分布”和真实峰形状分布像不像？

### 5.4 `count_weight=0.1` 不代表 count 不重要

它只代表：

> 在当前量纲下，如果把 profile loss 记作 `1.0`，count loss 只给 `0.1` 的相对权重更稳。

这是一种优化权衡，不是科学重要性排序。

### 5.5 `early stop` 不等于训练失败

它通常只是说明：

> validation 已经好几轮没变好，再继续跑大概率是在浪费 GPU。

### 5.6 `best.pt` 也不等于“最后一个 epoch”

训练会保存很多 epoch，但最终报告通常只用“选模规则挑出来的那一个”。
