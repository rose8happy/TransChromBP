# 学习材料索引

这是 `docs/learning/` 的唯一入口页。

- 维护 live 状态、family verdict、run 级事实的 canonical 文档不是这里，而是 [TRACKING.md](../../TRACKING.md)、[docs/experiments/registry.md](../experiments/registry.md) 和 [docs/experiments/runs.csv](../experiments/runs.csv)。
- 本目录只负责三件事：补概念、解释当前主线、提供阅读路径。
- 如果某份学习材料里的结论和 `registry/runs/closeout` 冲突，以后者为准。

如果你只想记一句话：

> 旧 BPNet / ChromBPNet 材料负责补基础；`docs/learning/` 负责解释当前模型与阅读路径；最新实验事实统一回到 `TRACKING.md`、`registry.md` 和对应 closeout 报告。

## 4. 按代码入口查

| 你想理解什么 | 先开哪个文件 | 为什么 |
|---|---|---|
| 当前主模型骨架 | `../../vendor/transchrombp/transchrombp/models/transchrombp.py` | 所有主线模型都从这里延伸 |
| Transformer 编码器 | `../../vendor/transchrombp/transchrombp/models/transformer_encoder.py` | `QKV`、attention、长程信息都在这里 |
| bias branch | `../../vendor/transchrombp/transchrombp/models/bias_branch.py` | `full/debiased` 语义离不开它 |
| loss / validation / selector 语义 | `../../vendor/transchrombp/transchrombp/training/train_ddp.py` | `compute_losses()`、`run_validation()`、`best_metric` 都在这里 |
| held-out test | `../../vendor/transchrombp/transchrombp/evaluation/evaluate_checkpoint.py` | 最终主数字从这里来 |
| Genos 接入 | `../../vendor/transchrombp/transchrombp/models/genos_adapter.py` | summary / FiLM / count 注入相关 |
| Caduceus 接入 | `../../vendor/transchrombp/transchrombp/models/caduceus_adapter.py` | token-level 在线融合相关 |
| foundation general adapter | `../../vendor/transchrombp/transchrombp/models/foundation_adapter.py` | cross-attention / residual 类挂载入口 |
