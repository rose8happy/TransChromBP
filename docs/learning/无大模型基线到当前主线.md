# 无大模型基线到当前主线

这份文档只看“不接入外部 foundation model”的主线，也就是：

> 只靠 DNA one-hot、项目自己的卷积塔 / Transformer / bias branch / readout，把模型一路推到今天的默认配置。

如果你只想知道“当前最好用、最值得读的几份模型是什么”，优先看这份。

## 1. 什么叫“不接入大模型”

在本项目里，“不接入大模型”不是“结构很简单”，而是：

- 不从外部 Genos / NT v2 / Caduceus 读取 frozen hidden states
- 不走 AlphaGenome API sidecar
- 输入仍然是本地 DNA 序列
- 主干仍然是项目自己定义的 `TransChromBP`

这条线的核心价值是：

1. 代码最稳定
2. 结果最干净
3. 和论文主线最一致

## 3. 为什么这几份模型最值得读

### 3.1 `Teacher V2 epoch_024`

它的重要性不在于“这是最复杂的版本”，而在于：

- 它第一次把 peak `count_r` 拉到 `0.8465`
- 同时把 peak `JSD` 压到 `0.3141`
- 而且 `full` / `debiased` 的 profile JSD 已经基本重合

换句话说，它是项目从“现象很多”走向“主线开始稳定”的起点。

### 3.2 `V2-full`

这条线回答的是一个更底层的问题：

> 如果别的东西都尽量不动，只问 Transformer 有没用，答案是什么？

现在这个答案已经很稳：有用，而且不是偶然。

### 3.3 `corrected B / center pool`

这是你今天最应该默认记住的模型名。

它之所以成为默认，不是因为“看起来最顺眼”，而是因为：

1. `center pool` 跨 seed 稳
2. `attention pool` 在第二个 seed 上失守
3. clean matrix 后的叙事也更容易和它对齐

### 3.4 `controlled L3`

如果你要回答“我们相对 official 到底有没有真优势”，现在最该看的不是旧 tutorial，而是 `L3 shared-region`。

它的重要性在于：

- 不是只在本地 validation 好看
- 而是在更受控的 external comparison 里也能站住

### 3.5 `no-bias`

这份结果容易被误读。

它不是在说：

> bias branch 没用。

它真正说明的是：

> 在当前 single-seed 边界补证里，去掉 bias branch 没有出现那种“立刻崩盘”的强证据。

所以它只能做 supplementary boundary，不是新的默认主线。

## 4. 为什么 `center pool` 会成为默认

这件事值得单独记住，因为它是很多配置文件的分界点。

当前更稳的理解是：

1. 它比 `attention pool` 更稳定，不靠个别 seed 撑结论。
2. 它更符合当前任务的局部中心窗口直觉。
3. 它和 `full/debiased` 的诊断逻辑更容易一起解释。
4. 它在后续 corrected-B、clean matrix、`L3` 这些口径下都没有变成短板。

## 6. 一句话结论

如果你之后只想记住一条主线，可以记成：

> 不接 foundation model 时，项目最值得看的不是“某个单独高分 run”，而是 `Teacher V2 -> V2-full -> corrected B / center pool -> controlled L3` 这条收口链；它说明主干 backbone、默认 readout 和 external evidence 都已经被逐层问清楚了，而 `no-bias` 只作为边界补证保留。
