## 三、已知问题与判断规则（避免重复踩坑）

- 自定义训练不要为 `bigWig` 单独安排“下载任务”；`chrombpnet pipeline` / `workflows/train_*` 会从 `bam/fragment/tagAlign` 现场生成。真正值得提前缓存的是 `hg38.fa(+.fai)`、`hg38.chrom.sizes`、`blacklist.bed.gz`、`folds.json`、可复用 `bias.h5`，以及仅在频繁生成 `nonpeaks` 时再额外缓存 `genomewide_gc`.
- AlphaGenome 当前适合做外部 black-box 对比与 variant/locus 分析，不适合把 API 输出拿来训练学生模型；如果后续要接，优先做 `1000s` 级别的小规模对照，不要直接设计成百万窗口扫描。
- 6000 上项目默认 `chrombpnet` 环境当前是 `Python 3.9.23`，与 AlphaGenome SDK 的 `Python >= 3.10` 要求不兼容；AlphaGenome 相关脚本应使用单独的 `Python >= 3.10` 环境，而不是直接污染现有训练环境。
- 6000 上可直接复用的 AlphaGenome 专用环境现为 `/data1/zhoujiazhen/bylw_atac/.mamba/envs/alphagenome`；pilot 脚本同时兼容 `ALPHAGENOME_API_KEY` 和官方 quick start 常见的 `ALPHA_GENOME_API_KEY` 两种变量名，但密钥文件仍应放在仓库外（当前使用 `~/.config/alphagenome/api.env`，权限 `600`）。
- `chrombpnet` 教程脚本中的 `chrombpnet_deepshap`、`chrombpnet_convert_html_to_pdf` 在当前环境里可能不存在；优先使用 Python 模块入口或补充 CLI 安装，而不是重复重跑前面步骤。
- `bias_qc` / `chrombpnet_qc` 历史失败里有一类原因是环境未激活（`modisco: not found`）；在 6000 上执行前必须先执行 `AGENTS.md` 中的环境激活命令。
- `foundation_models` 的 HF 下载目录里存在较多 `.lock` 文件（多在 `.cache/huggingface/download`），这不等于下载失败；是否可用以“最小加载验证”为准。
