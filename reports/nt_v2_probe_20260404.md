## Probe 结果

### 1. baseline 参考

- `encoded_only_auc = 0.868116`
- baseline `pred_logcount_r2 = 0.473401`
- baseline `pred_logcount_pearson_r = 0.750753`

这说明当前 `corrected-B / center-pool` baseline 在同一批 `valid` 样本上的区分能力已经很强。

### 2. NT 独立信号

最好的 `nt_only` 组合是：

- `layer_07__bins4_mean`
- `nt_only_auc = 0.742424`

这条结果高于 Genos 参考 `genos_only_auc = 0.6997`，说明：

- NT v2 确实携带独立的 accessibility 相关信号
- 而且这个信号更偏“早层 + 粗空间 summary”，不是最后一层的全局均值

### 3. 对 baseline 的互补性

最好的 `concat(encoded, nt)` 组合仍然是：

- `layer_07__bins4_mean`
- `concat_auc = 0.857792`
- 相对 `encoded_only` 的差值为 `-0.010324`

这意味着：

- NT v2 虽然有独立信号，但直接和当前 baseline 拼接后仍然拖累结果
- 它没有达到本轮设定的互补性门槛：`concat_auc >= encoded_only_auc - 0.005`

### 4. count 相关性

若只看 NT 对真实 `logcount` 的直接回归，最好组合是：

- `layer_14__bins4_mean`
- `nt_logcount_r2 = 0.106456`
- `nt_logcount_pearson_r = 0.530499`

这个数值本身高于 Genos probe 里 `count residual ridge` 的 `R²=0.0167 / r=0.1987`，说明 NT v2 对 region-level count 的**直接相关性**并不弱。

但若改成真正更重要的“对当前 baseline 残差的补充解释”，所有组合都失败：

- `nt_residual_r2` 全部为负
- 最好也只有 `-0.538603`

这说明：

- NT v2 没有在本轮 probe 中显示出“补 baseline 的 residual signal”
- 至少在当前最自然的 summary 读法下，它不像一个可直接提升现有模型的互补特征源

## 结论

### 技术 gate

通过。

- 当前 NT v2 目录在现有 `genos-1.2b` 环境里可以稳定、本地、GPU 化地加载和前向。
- 旧的 size mismatch 报错本轮未再复现。

### research gate

不通过。

原因分三条：

1. `nt_only` AUC 虽然过了门槛（`0.7424 > 0.73`）
2. 但最佳 `concat` 仍比 `encoded_only` 低 `0.0103`，未达到 `-0.005` 的互补性要求
3. 更关键的是，`residual` 回归全部为负，说明它没有对当前强 baseline 暴露出可直接利用的补充信号

## 下一步建议

- 不直接进入 tutorial A/B training integration。
- 若后续还想继续 NT 路线，优先考虑：
  - 只保留 `early-layer bins4 summary` 这一类高信息量读法
  - 先做更强约束的轻量 probe/adapter 设计，而不是直接把 NT hidden state 拼进当前 backbone
  - 把“token length = 355，不是 base-wise 2114”当作设计硬约束

一句话结论：

> NT v2 在当前任务上“有信号，但还不够互补”；它比 Genos 更像值得研究的候选，但本轮证据仍不足以支持直接进入 tutorial A/B training integration。
