# 动态 Loss 协调咨询问题总览

## 这份文件是什么

这份文件是本轮专题外发的主问题清单。

它的目的不是重复项目历史，而是明确告诉外部大模型：

- 这轮真正想回答的是什么；
- 它应该主动去搜哪些近年方法；
- 什么样的回答对我们最有价值；
- 哪些方向不值得它花主要注意力。

本轮默认优先级不是“最小改动、最快能跑”，而是：

> 在当前代码事实和证据约束下，优先判断最值得迁入的动态多 loss 协调方向。

---

## 1. 你应优先扮演什么角色

请优先把自己当作：

- 多目标训练 / loss 协调顾问
- 训练动力学评审者
- 最小验证实验设计顾问

请不要把自己首先当作：

- 论文润色器
- 泛泛讲多任务学习常识的科普作者
- 只会建议“小修权重”的保守工程师

---

## 2. 你必须主动做的前置动作

请默认使用你的搜索能力，主动检索近年的代表性方法和论文。

至少要自己判断并比较这些方向中的相关候选：

- 动态标量权重方法
- 基于梯度范数 / 梯度冲突的协调方法
- 基于不确定性或任务噪声的加权方法
- 任何专门讨论“训练目标与验证 / 最终指标错位”的方法

我们不要求你局限在某一篇文章，也不要求你只给“动态权重”这一类答案。

---

## 3. 你应该重点回答的 6 组问题

### A. 这个问题在我们框架里到底是什么问题

请回答：

1. 我们当前更像是在面对：
   - 多目标冲突
   - loss 尺度失衡
   - 结构耦合导致的伪冲突
   - selector 与最终指标错位
   - 还是以上几类的组合
2. 在 `full` / `debiased` 并存的结构下，应该把它们视为：
   - 并列主任务
   - 主任务 + 诊断目标
   - 主任务 + 训练约束
   - 或别的关系

### B. 哪类方法最值得优先考虑

请回答：

1. 结合近年的研究，最值得优先考虑的 3 类方法是什么？
2. 这 3 类方法里，哪些最适合我们当前代码结构？
3. 哪些看起来前沿，但在我们这里大概率只是“方法名好听”而非高价值迁移？

### C. 动态协调应该作用于哪些 loss 项

请回答：

1. 我们是否应该动态协调：
   - `full profile`
   - `full count`
   - `debiased profile`
   - `debiased count`
2. `debiased_count_weight` 当前为 `0.0`，它应继续保持关闭，还是应该进入协调体系？
3. distill loss 当前只是次级项；在主线方法判断里，应不应该先忽略？

### D. selector 和最终指标应不应该一起纳入设计

请回答：

1. `peak.loss_total` 与 `peak.profile_target_jsd_full_mean` 的不一致，应该被理解为：
   - 单纯选模口径问题
   - 训练目标设计不对齐
   - 或两者同时存在
2. 如果你设计新的 loss 协调方案，是否还应同步改 selector？
3. 是否存在比“继续最小化混合 loss”更合理的训练 / 选模配对？

### E. 哪些失败模式最值得警惕

请回答：

1. 哪类方法最可能进一步放大 bias-related shortcut？
2. 哪类方法最可能因为 `full/debiased` 非独立任务而产生误导？
3. 哪类方法最可能在 `count_fusion=logsumexp`、`learnable_scales=true`、`count_weight=0.1` 这些条件下失真？

### F. 最小但高信息量的验证闭环是什么

请回答：

1. 如果只允许做 `1-3` 个动作，应该怎么验证你最推荐的方法？
2. 每一步应该看哪些指标：
   - `peak profile JSD`
   - `peak count Pearson`
   - `full/debiased gap`
   - selector 稳定性
   - 还是别的东西
3. 什么结果算“值得继续”，什么结果算“应立即停表”？

---

## 4. 什么样的回答最有价值

最有价值的回答应该：

1. 先明确当前代码事实和 loss contract；
2. 再结合近年论文，给出有约束的判断，而不是泛泛总结；
3. 能区分：
   - 什么是值得动态协调的真实矛盾
   - 什么只是结构语义没说清楚
   - 什么只是 selector 错位
4. 对每条建议都说明：
   - 为什么可能有效
   - 为什么可能失败
   - 最小验证方式是什么
5. 最后压成 `1-3` 个高信息量动作。

---

## 5. 什么不值得你花主要注意力

请不要把主要注意力放在下面这些地方：

1. 重复讲“多任务学习一般怎么做”的入门常识。
2. 继续把旧的 `Transformer/bias` 连接误会当作主问题。
3. 不看代码语义，只凭论文标题推荐方法。
4. 只讨论训练 loss，不讨论 selector 和最终报告指标。
5. 一上来就建议重写整个训练框架，而不先给最小证伪方案。
