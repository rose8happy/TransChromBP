## 💡 核心知识点总结

### 1. 模型架构

```
输入 DNA序列 (2114 bp)
    ↓
[第一层卷积] 学习基本motifs
    ↓
[膨胀卷积 × 9层] 扩大感受野 + 残差连接
    ↓
    分支
   ↙  ↘
Profile  Counts
 预测     预测
```

### 2. 三大创新点

#### 🔹 膨胀卷积 (Dilated Convolution)
- **问题**: 普通卷积感受野小，无法捕获长距离依赖
- **解决**: 用膨胀卷积，感受野指数增长
- **效果**: 9层就能看到2000+ bp的范围

#### 🔹 残差连接 (Residual Connection)  
- **问题**: 深层网络难以训练，梯度消失
- **解决**: 跳跃连接，让梯度直接传播
- **效果**: 保留多尺度特征，训练更稳定

#### 🔹 双输出头 (Dual Heads)
- **问题**: 只预测一个值信息不够
- **解决**: 同时预测分布(Profile)和总量(Counts)
- **效果**: 两个任务互补，效果更好

### 3. 参数说明

```python
# 固定参数
conv1_kernel_size = 21     # 第一层卷积核大小
profile_kernel_size = 75   # Profile头的卷积核大小

# 可调参数
filters = 64               # 卷积核数量（通道数）
n_dil_layers = 9           # 膨胀卷积层数
counts_loss_weight = 5.0   # Counts损失权重
sequence_len = 2114        # 输入序列长度
out_pred_len = 1000        # 输出预测长度
```

### 4. 损失函数

```python
# Profile头: Multinomial NLL
# - 适合预测概率分布
# - 关注分布的形状

# Counts头: MSE  
# - 适合回归任务
# - 预测总信号强度

# 组合损失
Total Loss = 1.0 × Profile_Loss + 5.0 × Counts_Loss
```

| 概念 | 简单解释 | 举例 |
|------|---------|------|
| **DNA序列** | 由A、T、C、G组成的字符串 | "ATCGATCG" |
| **One-hot编码** | 把字母转换成数字向量 | A→[1,0,0,0] |

## 🔧 常见问题 FAQ

### Q1: 为什么输入长度是2114？
**A**: 这是经过优化的：
- 中心1000bp用于预测
- 两边各557bp提供上下文信息
- 保证中心区域有完整的感受野

### Q2: 为什么膨胀率是2的幂次？
**A**: 指数增长效率最高：
- 线性增长: 需要很多层才能达到大感受野
- 指数增长: 9层就能达到2000+ bp

### Q3: 什么时候需要调整参数？
**A**: 
- **数据少** → 减小 filters（防止过拟合）
- **序列长** → 增加 n_dil_layers（扩大感受野）
- **Profile不好** → 减小 counts_loss_weight
- **Counts不好** → 增大 counts_loss_weight

### Q4: 训练需要多长时间？
**A**: 
- GPU (V100): 1-2天
- GPU (RTX 3090): 2-3天
- CPU: 不推荐（太慢）

### Q5: 可以用于其他任务吗？
**A**: 可以！这个架构适用于：
- ChIP-seq信号预测
- RNA结构预测
- 蛋白质-DNA结合预测
- 任何序列到序列的任务
